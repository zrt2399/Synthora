# 贡献指南

我们欢迎任何形式的贡献，无论是报告问题、参与讨论、提交代码，还是提供建议。为了维护项目的健康发展，请您在贡献前仔细阅读并遵守以下规则。

## 行为准则

请务必遵守我们的 [行为准则](CODE_OF_CONDUCT.md)。我们致力于营造一个友好、互相尊重、富有建设性的社区环境。

## 如何贡献

### 报告问题 (Issues)

- 在提交问题报告前，请先搜索现有的 Issues，确保您的问题尚未被报告。
- 请详细描述您遇到的问题，包括：
    - **Synthora 版本:**
    - **操作系统:**
    - **复现步骤:**
    - **期望行为:**
    - **实际行为:**
    - **截图或录屏 (如果适用):**

### 提交代码 (Pull Requests)

我们非常欢迎您为项目贡献代码。在提交 Pull Request (PR) 前，请注意以下几点：

1.  **Fork & Clone:**
    - 首先，Fork 本项目到您的个人仓库。
    - 然后，将您的仓库 Clone 到本地。

2.  **创建分支:**
    - 从 `main` 分支创建一个新的特性分支，分支命名请清晰地描述其功能，例如 `feature/new-control` 或 `fix/button-style-bug`。

3.  **编码规范:**
    - **代码风格:** 请遵循项目现有的代码风格和格式。我们主要遵循标准的 C#/.NET 编码规范。
    - **命名约定:**
        - 控件 (Controls): PascalCase, e.g., `MyNewControl`
        - 转换器 (Converters): PascalCase, e.g., `BooleanToVisibilityConverter`
        - 附加属性 (Attached Properties): PascalCase, e.g., `Grid.Row`
        - 样式 (Styles): 在 XAML 中，Key 通常使用 `x:Key="MyControlStyle"` 的格式。
    - **注释:** 对于复杂或不直观的代码逻辑，请添加必要的注释。

4.  **XAML 规范:**
    - 保持 XAML 文件的整洁和可读性。
    - 属性请按照 `x:Name`, `Classes`, `Width`, `Height`, ... 的顺序排列。
    - 尽量使用 `ThemeResource` 或 `StaticResource` 来引用颜色和画刷，以支持主题切换。

5.  **提交 PR:**
    - 确保您的代码能够成功编译，并且没有引入新的警告。
    - 在 PR 的描述中，请清晰地说明该 PR 的目的和所做的更改。如果修复了某个 Issue，请使用 `Closes #issue_number` 的方式关联。
    - 我们会尽快审查您的 PR，并提出修改建议或合并它。

## 项目结构概览

-   `/Controls`: 存放所有自定义控件。
-   `/Converters`: 存放值转换器。
-   `/Extensions`: 存放扩展方法。
-   `/Styles`: 存放控件的默认样式和主题。
-   `/Accents`: 存放主题颜色定义。
-   `/Strings`: 存放本地化字符串。

## 编译与代码完整性要求 (Compilation & Integrity Rules)

- **强制静态检查：** 在输出最终代码建议之前，你必须在内部执行一次“静态编译检查”。仔细核对括号是否闭合、类型是否匹配、变量是否已声明、上下文逻辑是否连贯。
- **严禁代码省略 (No Placeholders)：** 提供的代码必须是完整的、可直接复制粘贴并成功编译的。绝对禁止使用 `// ... existing code ...`、`// 在这里添加逻辑` 或 `/* 剩余代码 */` 等会导致编译直接报错的省略写法。
- **处理依赖关系：** 如果你的代码引入了新的类、接口或扩展方法，**必须**在代码顶部显式提供所需的 `using` 命名空间。如果涉及到新的 NuGet 包（如特定版本的 MQTT 库），必须单独列出需要安装的依赖。
- **上下文兼容：** 严格遵循 C# 11/12 语法标准和 Avalonia 11 框架特性。在调用 API 时，务必确认该 API 在当前版本中未被弃用，以防止产生 CS0618 (已过时) 或更严重的编译错误。
- **自我验证提问：** 如果你不确定某段代码是否能在此项目中编译通过（例如不确定 `IMqttService` 的具体接口定义），**必须先向我提问获取上下文**，而不是盲目猜测并给出错误代码。

- 感谢您的贡献！
